**R17 engineering changes in effect (different from R16):**
- **Tolerance-based threshold resolution** in `_check_threshold` (engine_v2.py): R16's margin-based comparison is preserved, but now uses a `tol = max(1e-9, 1e-9 * |threshold|)` floor so FP-ULP-sized margin differences (~1e-15) from ordering of P1's vs P2's `_apply_propagation` resolve as draws, not phantom P1 wins. Fixes the R16 bug where 4 sim teams hit cases of P2 having higher true margin but FP noise giving P1 the win.
- **`_fix_consistency` now mirrors generator's moore+surround → grid downgrade** (`evolution/operators_v2.py`). Closes the R16 hole team-17 located.
- **Threshold parity-shift**: generator and mutator both perturb generated/mutated thresholds by ±0.5 × stone-contribution, breaking the alternating-game tempo trap that fitness was selecting for in R16 (multiple teams flagged 67-87% P1 winrate in greedy probes).
- **Seat-balance floor lowered 0.2 → 0.1, hard-zero when seat_bal < 0.1**. R16's threshold-race winner had documented severe P1 bias (team-18: greedy 5/5 P1; team-19: 73% P1 across opening sweep) yet kept rank 1. R17's stricter gate kills these games — both R16 winners (`c6bb58075520`, `8d12c8b92b71`) seeded into R17 got GE=0 from this gate firing.
- **Sierpinski substrate added to topology pool** with the constraint that fitness selects against `sierpinski + threshold` (gated soft) and `sierpinski + CA` (gated soft). Audit mode lets violators train so post-run analysis can validate the rule. R17 audit found violators score significantly worse — soft rules will fold to hard for R18.
- **Substrate-novelty multiplier**: GE composite gets +5% for non-grid topologies, capped at 1.0. Helps non-grid games stay competitive when depth/diversity are comparable.
- **Hybrid place+move action games** are now well-represented in R17's top 10 — multiple top games allow stones to be moved to adjacent cells in addition to placement. Action space scales as `total_cells + 1 + total_cells × max_degree`, so for 3D moore (max_degree=26 in 3D) the action space is ~1700, roughly 26× larger than place-only. Reading the rules carefully matters.

**Known engine quirks and prompt caveats (still present from earlier runs):**
- Action N-1 (where N = num_actions) is **pass**. Two consecutive passes end the game as a draw.
- For move-enabled games, action layout is: `[0..total_cells-1]` = place actions, `total_cells` = pass, `[total_cells+1 .. total_cells + total_cells×max_degree]` = move actions encoded as `from_cell × max_degree + neighbor_idx`.
- On **torus** topology, influence propagation wraps around edges, but **custodian capture walks do NOT wrap** (they stop at the hard axis boundary). Real engine behaviour, may mislead positional reasoning.
- `play_helper.py show` does not display the effective influence-value field. If a game uses propagation, drop into `.venv/bin/python` and load `engine.board_values` (numpy attribute, not method) to read true board values.
- Action IDs for placement are `coords_to_cell` ordering; for 2D this is `y * axis_size + x`, for 3D it's `z * axis^2 + y * axis + x`. Typos are easy — double-check.
- `play_helper.py rules` may mislabel topology (Moore mislabeled as "von Neumann" historically). Verify by reading `topology.py` directly if uncertain.
- `engine.get_legal_actions()` returns `list[int]` of legal action IDs (NOT a 0/1 mask). Iterate over it directly.

=== TEAM COMPOSITION ===

Your team consists of three roles. Handle them as distinct passes (one agent playing all three sequentially is fine; acknowledge the seat-identity bias in Phase 3):

1. Player 1 — plays the game from the first-player perspective

Run the phases in order. Do not skip the Novelty Adversary phase.

=== PHASE 1 — RULE COMPREHENSION ===

Read the game's rules and topology. Produce a plain-English summary covering:
- Board structure (dimensions, axis size, topology type) — **R17 games are mostly 3D; visualise the spatial layout carefully**
- **Turn structure (alternating or simultaneous — call this out explicitly and explain the collision-resolution rule if simultaneous)**
- Action types (place, move, or both) — **for place+move hybrids, note both placement constraints AND movement constraints; the action space is ~26× larger than place-only**
- Placement/movement constraints
- Capture or cellular-automaton dynamics (if any) — for CA, note how many transition entries are non-trivial (non-identity)
- Propagation or influence effects (if any) — note radius, strength, decay
- Win condition (with specific threshold values; for connection wins, **note which axes P1 vs P2 must connect — they're typically asymmetric in 3D**)

Flag any rules that are degenerate (e.g. "player 1 always wins by playing action 0 repeatedly", "game ends in ≤5 moves with a forced win", "capture rule is inert", "most CA rules never fire", "threshold is unreachable so the game resolves by double-pass draw").

=== PHASE 2 — STRATEGIC PLAY ===

Play 3 full games between Player 1 and Player 2 agents.

- **Every move MUST be engine-verified.** Run each move through `play_helper.py` (alternating) or `sim_play_helper.py`/`step_simultaneous()` (simultaneous) before committing. The engine is authoritative.
- **For 3D games, build mental coordinate maps.** Cell index = `z*axis² + y*axis + x` for 3D. Don't conflate with 2D layout.
- **For place+move hybrids**: explicitly track which existing stones could move where. Move actions sometimes dominate; missing them is a Phase-2 quality issue.
- **Separate the reasoning.** When playing Player 1, do not read Player 2's reasoning until Player 1's move is committed.
- **Simultaneous games**: both players' moves must be submitted in the same step. Track collisions explicitly.
- **In game 3, swap seats.**
- **Time budget per game.** Budget ~25 minutes. If a game hits max_turns and is still contested, STOP and note non-convergence.
- **Role-switch discipline.** Finalize the previous role's reflection in writing before switching.

For each move:
- Explain what you are considering
- State your chosen move and the reasoning (1-2 sentences)
- Predict the opponent's likely response

After each game, each player reflects:
- What strategy did I use?
- What would I do differently?
- Did my opponent surprise me?
- Did the endgame reach the stated win condition, or resolve via double-pass draw / timeout?

After all 3 games, each player writes a short "strategy guide" for their side.

=== PHASE 3 — STRATEGIC ANALYSIS (joint) ===

Player 1 and Player 2 discuss:
- Are there distinct viable strategies, or does one approach dominate?
- Is there meaningful counter-play?
- Is there short-term vs long-term tension?

- **First-mover advantage:** for alternating games, did P1 dominate? For simultaneous games, did the mechanic actually eliminate it? Quantify by seat-swap results.
