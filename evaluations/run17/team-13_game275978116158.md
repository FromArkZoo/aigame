## PHASE 1 — Rule comprehension

### Board
- 3 dimensions, axis_size 4 → **4×4×4 = 64 cells**.
- Topology: `moore` in 3D. Confirmed by inspecting `topo._neighbors`: corners have 7 neighbours, edges 11, faces 17, interior cells 26. **Max degree = 26.** A "step" in moore-3D is any of the up to 26 cells whose Chebyshev distance is 1 (i.e. each coordinate differs by ≤1).

### Turn structure
- **Alternating**, 1 piece per turn. P1 moves first.

### Action types — place + move hybrid
- `action_rule.action_types = ["move", "place"]`, `move_constraint = "adjacent_empty"`.
- Action layout (verified):
  - `0..63` = place at cell `c` where `c = z·16 + y·4 + x`.
  - `64` = pass; two consecutive passes ends the game as a draw.
  - `65..1728` = move; `(a − 65)//26 = from_cell`, `(a − 65) % 26 = neighbour-slot index`.
- **Movement**: a player may pick up one of their own stones and drop it onto an empty moore-neighbour cell. Capture re-fires from the destination cell after a move (engine `_handle_movement` calls `_apply_captures(to_cell)`), so a move is functionally equivalent to "pick up + place on adjacent empty". Verified live with engine: P1's `move (1,1,1)→(1,2,1)` clears 21 and fills 25 with no piece-count change.
- Placement constraint: target empty, anywhere; `first_move_anywhere = True`.

### Capture — outnumber, threshold 2
On placement (or move), each enemy stone that is moore-adjacent to the placed/destination cell is removed if the count of friendly stones in its moore-neighbourhood is ≥ 2 (the just-placed stone counts).
- **3D moore + threshold 2 is *brutal*.** Every stone — even a corner — has 7+ neighbours. Two friendlies anywhere in the 3×3×3 cube around an enemy is enough to remove it.
- Capture only fires from the actively-placed stone; an enemy stone *already* surrounded does **not** re-trigger passively. So a player can voluntarily wedge into a fully-surrounded cell and survive for one tempo before it's swept on the opponent's next turn.

### Propagation
- `prop_type = "none"` — propagation rule is a no-op despite `radius/strength/decay` being populated. **`board_values` stays at zero throughout.** Threshold-style win checks therefore never fire (only relevant for `condition_type == threshold`).

### Win condition — connection
- `condition_type = "connection"`, `threshold = 0.5`, `target_dimension = 1` (P1 connects **y-axis**: y=0 face ↔ y=3 face), `target_dimension_p2 = 0` (P2 connects **x-axis**: x=0 face ↔ x=3 face).
- BFS over each player's stones using moore adjacency. Connection requires a path with each consecutive pair Chebyshev-1.
- Both players completing on the same tick → draw (R16 fix). In an alternating game this can only happen if the current move's capture removes the mover's own connection-blocking enemy and simultaneously completes both — extremely rare.
- `max_turns = 100`. If reached, majority of pieces wins (else draw).

### Degeneracy flags
- **Connection requires *exactly* 4 stones minimum.** Axis_size = 4, Chebyshev step = 1, so any chain from coord 0 to coord 3 needs cells at every integer coord 0,1,2,3 along the target axis. Each consecutive pair can share x and z within ±1. So the **shortest possible connection is a 4-stone chain**.
- P1 has the move ordering 1,3,5,7,…; P2 has 2,4,6,8,… A pure race ⇒ P1 reaches 4 stones on **turn 7**, P2 on turn 8. **The game is structurally a forced P1 win unless P2 can force ≥1 P1 capture per 1 net P2 move spent.**
- Outnumber-threshold = 2 makes capturing a *single isolated* enemy require placing 2 stones around it — no faster than completing the chain.
- "first_move_anywhere=True" only matters if `placement_rule.constraint` excludes empty cells; here it's "anywhere" so this flag is inert.
- Propagation block is dead code (prop_type=none).

### Strategy guides

**P1 strategy**: open at the (1,1,1) anchor (or any interior cell). Always reply to a P2 stone whose Chebyshev distance from your anchor is 1 *with* a stone that is moore-adjacent to both your anchor and the wedge **and** lies on your target y-coord layer. The 3×3×3 cube around the anchor virtually guarantees such a "capture-and-advance" cell exists, so each P1 turn buys one chain step *plus* one P2 capture.

**P2 strategy**: structurally none of the 3 games revealed a winning P2 plan.
- *Race far*: loses by 1 tempo (Game 2).
- *Wedge near*: every wedge gets captured by a P1 advance stone (Games 1, 3).
- *Wall-block*: would need ~9 cells across y=2 (or x=2) plane; cost-prohibitive.
- *Outnumber-counter*: requires 2 P2 stones around one P1 stone. By the time P2 has 2 stones placed, P1 has captured at least one — so this is net negative tempo.

### Move-action usage
**Zero move actions** were chosen across all 3 games. Optimal play in this 4×4×4 race never leaves any of P1's 4 chain stones in a position where moving it improves connection — the chain is built up additively from y=1 and a move would empty an existing chain cell. Move is a **vestigial action** for the strategic core. (It can be useful tactically if both stones cleared a single-stone capture trap, but no game reached that state.)

## PHASE 3 — Joint strategic analysis

**Distinct viable strategies?** No. P1 has a single dominant template (interior anchor + capture-on-path advance). P2 has no winning template.

**Counter-play?** Within P1's plan, choosing the capture-advance cell sometimes has 2–3 equally good options — minor tactical width. Outside that, no.

**Short- vs long-term tension?** Almost none. The game is concluded by turn 7. There is no opening / midgame / endgame distinction.

**Emergent concepts?**
- *Tempo* (P1's 1-move lead is decisive).
- *Capture-as-advance*: a mild emergent property — P2's wedges are always inside P1's chain pocket, so capturing them is free progress. This is genuinely 3D-flavoured because 2D versions of the same idea have fewer "intersection cells".
- *Sub-volume control*, *path-routing*, *initiative*: not really — the chain is so short and the board so small that path-finding is trivial.
- *Ko fights*: none observed; outnumber capture needs setup, no immediate re-capture possible since the original mover's stone would still be there.

**Does 3D add depth?** *Marginally negative*. With axis_size 4 and moore adjacency, the connection is only 4 cells and routing is overdetermined — moore's diagonals make the chain too easy. A 2D moore (chess king) board with the same dynamics would require a longer chain (5×5 or 8×8) to be interesting; in 3D it caps at 4. **A 2D Hex variant (with some capture) would have at least as much depth and likely more — moore's 26-neighbour generosity in 3D is the *opposite* of what depth wants.**

**Topology mattering?** Yes, in the wrong direction — moore makes capture and connection *too* easy. Replacing moore with grid (von-Neumann) in 3D would require coords to differ by at most 1 along *one* axis per step, making a 4-cell y-axis chain essentially a single column. That would *also* be trivial. The substrate is fighting the game design.

**Move-actions used?** **0 / 0 / 0** across the 3 games. ~26× larger action space, none of it useful in optimal play. Vestigial.

**First-mover advantage**: 3/3 P1 wins on turn 7. With seat-swap discipline (Game 3 played from P2's perspective with P2 playing best counter-attempts found), P2 still lost. By the symmetry argument (P1↔P2 + axis swap y↔x), if P2 had moved first they would win identically — so **the game is fundamentally first-mover-decided**. Engineering training runs corroborate: the 6 RL runs showed wildly oscillating final-winrates (0/0.5/1/0.5/0/0.5) — typical of an unbalanced game where PPO ends up at whichever side initiative landed on per training run.

## PHASE 4 — Novelty adversary

The Novelty Adversary argues the game is not novel. Charges:

(a) **3D Tic-Tac-Toe / Qubic / Sogo (4×4×4 connection)**. Qubic is *exactly* 4×4×4 with a 4-in-a-line connection win. This game's 4-stone chain is essentially a Qubic line *but allowed to bend at every step* (because moore adjacency permits any king-move). So this is "Qubic with crooked 4-lines and removable stones". Qubic is a known forced first-player win — and so is this.

(b) **3D Hex / Y / Havannah generalisations**. Hex is connection-of-opposite-faces. Adding a 3rd dimension trivialises Hex's "no draws" property (well-known). 3D Hex has been studied (Yamaguchi, "3D-Hex"). The custodian/outnumber capture overlay is the only departure.

(c) **Go x Hex hybrids (Connection Go)**. R8's champion was 2D Connection Go. R17's grand champion `44f6630277b3` is 3D Connection Go with custodian. **This game is the cousin: 3D Connection Go with *outnumber* and a smaller "place+move" twist**. The mechanic family is shared, the differences are second-order.

(d) **Outnumber-capture games**. Lines of Action uses outnumber-style mass-vs-mass. Quoridor doesn't capture but is a connection game. There's no published "moore-3D + outnumber + connection" that I can locate, so the *exact combination* is novel.

(e) **Coordinate transformation reskin?** The 4×4×4 moore graph is isomorphic to a 64-vertex graph with extremely high mean degree (~17.5). Any sufficiently-dense graph + connection rule + capture is structurally similar.

(f) **Expert transfer**: a Qubic / Connection-Go / 3D-Hex player transfers immediately. Within 5 minutes they would discover the 4-stone race and the capture-on-path tactic.

### P1/P2 rebuttal
- The capture-and-advance dovetail (every wedge capture *is* a chain step) is novel-flavoured — it doesn't quite happen in Qubic (which has no capture) or in Lines of Action (which has no faces-to-connect).
- The fact that move actions exist but are never useful means the bigger action space adds nothing — so the "place+move" hybrid is **not** rebuttal material.
- The 3D moore + outnumber-2 combo creates an abnormally aggressive capture environment, but the small board + axis_size=4 chain length so dominates that this novelty is mostly cosmetic.

**Overall novelty**: at the *combination* level the game has not been published as far as I can find. At the *play-experience* level it is "Qubic with a side dish of capture-on-path", which a Qubic player would recognise instantly.

**Novelty score: 4/10** — combinatorially distinct, experientially familiar.

**Closest known-game analog**: Qubic (4×4×4 line connection) crossed with 2D Connection Go. Differs only by allowing chains to bend at every step (because moore) and by adding outnumber capture.

**Killer flaws**:
- **Forced P1 win on turn 7** from any opening (verified 3/3, plus structural analysis).
- **Move actions are vestigial** — 0/0/0 used; ~1664 of 1729 actions are dead weight in optimal play.
- **Outnumber threshold = 2 in 3D moore is too aggressive**: a single P1 advance stone always sits at the moore-intersection of the anchor and the wedge, so wedges are always 2-for-1 disadvantaged.
- **axis_size = 4 + moore = 4-stone connection** is the absolute minimum chain length, leaving zero room for strategic depth.
- Propagation block is dead code (prop_type=none); a second of the rule struct is doing nothing.

**Best quality**: the *capture-and-advance dovetail* — when P2 wedges adjacent to (1,1,1), there is *always* a P1 cell that simultaneously captures the wedge and advances P1's chain. This is a real (if minor) 3D-moore artefact that doesn't show up cleanly in 2D analogues.

**3D structural contribution**: **negative**. moore-26 in 4³ shrinks the minimum-chain length and saturates capture density. Flattening to 2D (e.g. Hex-13 with same outnumber rule) would give *more* depth. The 3D shift R17 selected for is here serving substrate-novelty multiplier, not gameplay.

**Improvement idea (single-rule)**: raise `outnumber threshold` to **4** (or downgrade topology to 3D-grid). Threshold=2 with 7–26 neighbours per cell is overpowered; capture happens on *any* second adjacency, so wedges die instantly. Threshold=4 would force real flanking patterns and prevent the capture-and-advance dovetail, restoring tempo to the wedge attacker. *Or*: increase axis_size to 5 (extends minimum chain to 5 stones, gives P2 a real chance to disrupt mid-chain).
