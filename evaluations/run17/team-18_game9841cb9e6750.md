- `max_turns = 100`. If not reached, stone-count majority decides at max_turns. Empirically, threshold is hit by turn 19–25; max_turns is irrelevant.

### Plain-English Summary
**"Race to accumulate 42 points of influence on cells you own, on a 4×4×4 cube. Stones radiate ±1.4 (decaying ×0.47 per Manhattan step) within radius 2. First to threshold wins. Captures essentially impossible due to 3D connectivity."**

## PHASE 3 — STRATEGIC ANALYSIS (joint)

### Distinct strategies?
- **Effectively one viable strategy:** build a tight 2×2×2 corner cube as fast as possible, then extend with cells that maximise (d=1 friendly count) + 0.47 × (d=2 friendly count).
- Sub-strategies (corner-cube vs central-cluster vs edge-line) all converge — corner cube dominates because (a) board boundaries reduce enemy contact and (b) the cube has 12 d=1+d=2 internal pairings, near-optimal.
- **Contact play is a Pareto-loss for both** — Game 2 confirmed: contact halves score growth but doesn't change the winner.

### Counter-play?
- **Essentially none for P2.** Best P2 disruption (multi-purpose attack at T18) closes race by ~0.6/turn; P1's tempo is +5/turn. Asymmetric.
- Capture is theoretically possible but practically impossible (need 6+ adjacent enemy stones in 3D 6-conn; budget is 10 stones total per side).

### Short-term vs long-term tension?
- **Almost none.** Each move's marginal contribution is mostly determined by local d=1/d=2 friend count. There is no "sacrifice now for territory later" — every move's value is myopically computable.
- The only tension is "build outward vs disrupt opponent" and disruption is dominated by build except in the very last turn before opponent crosses threshold.

### Emergent concepts?
- **Tempo:** the only concept that matters. P1 has it from move 1.
- **Cluster density** (d=1 vs d=2 fillrate) — interesting geometry, but converges to "fill corner cube first".
- **No territory, no influence-projection, no ko, no liberty-based fights.** The surround rule is dead config.
- **No 3D path-routing or sub-volume control** — these would matter for connection wins, but this is threshold.

### Does 3D matter?
- **Marginally.** The 3D substrate adds 6-connectivity (vs 4 in 2D), which makes capture even harder than 2D Go. Influence dynamics are similar to 2D but with more cells per radius shell (6 vs 4 at d=1, 12 vs 8 at d=2).
- **The same game on a 2D 8×8 grid with influence radius 2 would play essentially identically** — race to corner cluster, first-mover wins. The 3D doesn't enable any "out-of-plane" tactic because there's no connection requirement.
- **3D structural contribution: very low.** This game is a 3D re-skin of a 2D-style threshold-influence race.

### Topology?
- Grid was the natural substrate here. Hex/torus would change neighbour-count, but the dynamics ("cluster fast, race to threshold") would persist.

### Move-enabled? 
- N/A — place-only (action_types = ['place']). The 5/10 R17 hybrid trend doesn't apply to this game.

### First-mover advantage (quantified)
- **3 of 3 games went to P1, all by threshold cross on P1's odd-numbered turn.** P1 score at win: 43.57 / 43.85 / 43.57. P2 final: 37.64 / 36.60 / 37.64.

## PHASE 4 — NOVELTY ADVERSARY (mandatory)

### Adversary's case: this game is NOT novel

(a) **Catalog comparison:**
   - **Reversi/Othello**: place stones for territorial advantage. No, this lacks the flip mechanic.

   - **Tumbleweed**: each cell has visibility-line-count value. No, that's a different value function.
   - **3D Tic-Tac-Toe / Qubic**: line-completion in 4³. No, no line condition.
   - **3D Hex / Connect6**: 3D connection win. No, this is **threshold not connection**.
   - **Influence-based Go variants** (e.g. "Field Go", "Atomic Go"): place stones, accumulate influence/score by formula. **Yes, very close.** This is essentially "Score-Go without territory + Gaussian-decay influence + race to threshold instead of comparison".
   - **Reskin claim:** This is **"Threshold-Influence-Race on a small N-D cube"**. The mechanic would behave identically on a 2D 6×6 grid (matching cell count) or a 1D line of 64 with longer radius — just a race to corner-cluster.

(b) Not CA-based.

(c) Not simultaneous.

(d) **3D + connection variant?** No — this is 3D + threshold, and threshold doesn't use the 3D structure (no axis-asymmetric win condition, no path-routing). The R17 narrative (3D Connection Go rediscovered) does NOT apply here — this game is in the *threshold* family.

(e) **Coordinate-transform reskin:** As noted — equivalent to a 2D 6×6 (36-cell) influence-race game.

(f) **Expert transfer:** A Go expert sees nothing recognisable (no liberties to track in practice), but a Reversi player would understand "place to maximise positional value" within minutes. **Transfer time: ~30 min for a numerical-board-game player.**

### Rebuttal (P1+P2 joint)

- **The threshold race is structurally first-mover-dominated**, in a way standard Go and Hex are not (those have known balance via komi/swap). This game has neither.
- **Best moment to argue novelty:** The interaction between cube-cluster geometry and influence dilution at contact zones (Game 2 T1–T8) created some interesting trade-off math, but the optimal strategy converged back to "extend outward" by T9 — no novel emergent dynamic.

### Novelty score: **3/10**

It's a 3D re-skin of a threshold-influence race game. The 3D substrate adds nothing the 2D version wouldn't. Capture is dead. No emergent dynamics beyond cluster geometry.

### CLOSEST KNOWN-GAME ANALOG
**"Influence-score race on a small grid"** — closest fit is a stripped-down hybrid of (a) Reversi (positional value of cells) without the flip mechanic, (b) Go without territory or active capture, with (c) a hard threshold replacing comparative scoring. No commercially-published direct analog, but the dynamics are well-trodden in threshold-race game-design literature (e.g. Gungo without the simultaneous play; "Field Go"). Distinguishing feature: 3D, but 3D adds nothing strategically.

### KILLER FLAWS
1. **Severe first-mover advantage** — 3/3 P1 wins in optimal play; tempo gap is structurally one move and the threshold doesn't bridge it.
2. **Capture rule is dead config** — surround in 6-conn 3D in a ~20-turn budget cannot fire.
3. **Surround-threshold parameter (1) is dead config** — surround logic ignores it.
4. **`target_dimension`/`target_dimension_p2` are dead config** for threshold wins.
5. **R17 threshold parity-shift failed** — 42.08 still falls in P1's winning bracket.

### BEST QUALITY
The cluster-geometry math (per-cell value as function of friend/enemy counts at d=1, d=2) makes it a clean teaching example of "influence-race on a lattice". The trained PPO converges quickly (ELO 2593, vs-random 0.92–0.96), suggesting the policy space has a single basin of attraction.

### 3D STRUCTURAL CONTRIBUTION
**Negligible.** The same dynamics would hold on a 2D 6×6 board (36 cells, similar effective volume) or a 4×4×4 with influence radius 2. There is no use of axis-asymmetry, no 3D path-routing (no connection win), no sub-volume control. **This is a place-only threshold game that happens to live on a 3D substrate.** R17's headline "3D shift discovery" claim does not apply here — this game is in the threshold family, not the connection-with-mobile-stones family that defines the R17 champion.

### IMPROVEMENT IDEAS
- **Replace threshold with comparative-margin** + penalty for first-crosser (komi-style): would address the seat imbalance directly.
- **Increase capture potency:** drop to outnumber capture with threshold 2, so a 2-friendly-attacker cluster can capture an isolated stone. This would create a real attack/defence axis.
- **Reduce radius to 1:** the d=2 contributions create the smooth race; cutting them forces sharper local fights and breaks the "build corner cube fastest" autopilot.
