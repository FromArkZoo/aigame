# Team-15 Evaluation — Game 275978116158 (R17, rank 3)

## Phase 1 — Rule Comprehension

- **Board:** 3D **moore** topology, axis_size 4 → 4×4×4 = **64 cells**.
  - Cell index = `z*16 + y*4 + x`.
  - Each interior cell has up to 26 neighbors (all cells within Chebyshev distance 1, excluding itself); boundary cells fewer.
- **Turn structure:** alternating, 1 piece per turn.
- **Action space:** 1729 actions = 64 place + 1 pass + 64×26=1664 move actions. Move encoded `64 + 1 + from_cell*26 + nbr_idx`. **Hybrid place+move; move_constraint = `adjacent_empty`** (move a stone to an empty moore-adjacent cell).
- **Placement:** anywhere empty. First-move-anywhere = true.
- **Capture:** **outnumber, threshold 2.** When a player places/moves at cell C, every enemy E moore-adjacent to C is captured if ≥2 of the placing player's stones are moore-adjacent to E (including the just-placed/just-moved stone).
- **Propagation:** `none` (the strength/decay/radius parameters are inert).
- **Win condition:** **connection** with asymmetric axes — **P1 must connect dim 1 (y-axis): y=0 face ↔ y=3 face**. **P2 must connect dim 0 (x-axis): x=0 ↔ x=3.** Connection is BFS via moore-26 adjacency restricted to own stones. `max_turns = 100`.

**Degeneracy / soft-rule check:**
- No degeneracy in win condition — the asymmetric face assignment (y for P1, x for P2) is non-trivial.
- Propagation parameters are stored but `prop_type=none` makes them dead — fine.
- **Move action space is largely vestigial** in practice (see Phase 2): with 4-stone connection target and ~7–9-turn games, no game required a move action.
- 4-stone moore-connected path is the win primitive. With moore-26 connectivity, ANY 4-tuple of stones differing by 1 in y per step (and within ±1 on x and z) connects faces. So strategies are very compact.

### Strategy guide

**Player 1 (connect-y):**
1. Pick a y-column at a z-layer ≥1 step away from both z-faces (z=2 ideal on axis 4).
2. Each y-step uses moore redundancy: a stone at (x,y,z) with 26 neighbors gives many y-extension options.
3. Avoid column-on-z=0 or column-on-z=3 — vulnerable to a perpendicular row at the corner.
4. If a y=k stone is taken/threatened, an "uncapturable replacement" exists if some y=k cell is Chebyshev≥2 from every existing enemy stone.
5. Only 4 placements needed — race is the dominant frame.

**Player 2 (connect-x):**
1. Without capture, you lose the race by 1 tempo. Capture is mandatory.
2. The "row trap" — three collinear stones in a low-y, low-z corner — is the canonical capture: it punishes any P1 column built at the parallel-adjacent (x,?,?) for x ∈ {anchor−1, anchor, anchor+1} with z ∈ {0,1}.
3. If P1 plays away from the row's reach (z=2, z=3), pivot to a different row OR engage with double-purpose stones (cells adj to 2+ P1 stones).
4. Move actions in opening don't help — you have too few stones.

## Phase 3 — Joint Strategic Analysis

- **Distinct viable strategies?** Two clear ones: (a) "tight column" race (works for P1 if z is chosen carefully), (b) "row trap" capture (works for P2 if P1 cooperates by stacking parallel-adjacent). Counter-play to (b): P1 picks a z-layer ≥2 away from any face. The strategy space is shallow — beyond these two patterns, choice doesn't matter much.
- **Counter-play:** Yes but binary. The trap either works or doesn't based on layer choice.
- **Short-term vs long-term:** No long term. Games converged in 7–9 turns; max_turns=100 is irrelevant. Tactics ≈ strategy.
- **Emergent concepts:**
  - **Parallel-adjacent capture geometry** — moore-26 + outnumber=2 makes 3-in-a-row enemy stones a pair-capturing weapon.
  - **Uncapturable cell** — any empty cell with 0 enemy moore-neighbours is safe for one full opponent turn.
  - Tempo accounting: each capture flips 2 tempi. One capture decides games.
- **Does the 3D substrate add depth?** Marginally. The 3D "escape lane" (P1 builds at z=2 to dodge a z=0 row) is genuine, but the same idea exists in 2D moore: column at y=k still has parallel-adjacency only to rows at y∈{k−1,k,k+1}. **The game would compress to 2D moore-8 with very similar dynamics.** 3D mostly adds redundancy for the connection, not new ideas.
- **Topology matters:** Moore (26-nbr in 3D, 8 in 2D) makes diagonal capture lethal. On 3D vN-grid (6-nbr), this game would be much slower (no diagonal capture, longer chains). The chosen topology is essential to the dynamics.
- **Move actions usage:** **Across 3 games (24 total turns), zero move actions were taken.** With games ending in 7–9 turns and a 4-stone target, no player ever has slack to relocate. The move action space (1664 of 1729 actions) is **vestigial** here. Move-induced captures are theoretically interesting but never triggered in practice.
- **First-mover advantage:**
  - Pure race math: P1 wins by 1 tempo (4 placements at T1,3,5,7 vs T2,4,6,8).
  - Capture flips this when P1 plays a parallel-adjacent column.
  - 3 games: P1 won 2, P2 won 1. Game 1 was a "P1 plays correctly" win. Game 3 was an "informed P1" win. Game 2 was "P1 plays naïvely" → P2 wins. So **P1 has the win when both sides play with knowledge of the row trap.**
  - The R17 GE pipeline reports `seat_balance` ≥0.1 was required; this game cleared that gate so its training-time imbalance was inside the soft window. With informed play it skews further to P1.

## Phase 4 — Novelty Adversary

**Adversary:** "This is Bridg-It in 3D with a Reversi-style capture bolt-on."
- Bridg-It (1958, Gale): two players race to connect opposite sides of a grid; no capture; place-only. The connection-race core is identical. The 3D moore lattice generalises Bridg-It's grid in a topology-preserving way.
- Reversi/Othello: outnumber-style capture (custodian flip in Othello, but the conceptual cousin "your stone surrounded by 2 enemies → flip" is well-explored).
- Stitching them together does not create a new strategic primitive — both halves transfer.

**Other comparables:**
- **3D Hex / Y-game on 3D lattice:** connection on 3D lattice exists in math literature (e.g., the "3D Hex" / "Higher Dimensional Hex" papers discuss winnability). No capture there.
- **Qubic / 3D Tic-Tac-Toe (4³):** different goal (line of 4), wrong family.
- **Lines of Action:** 2D, place-then-move; movement is essential (here it's vestigial). Different.
- **Connection-with-capture games:** Slither (2D place + move stones, surround-capture, connection win) is the closest known cousin. Slither is 2D and uses different capture (group surround).

**Adversary rebuttal from Phase 2:**
- The **row-trap double-capture geometry** observed in Game 2 is *not* a Bridg-It concept and not really a Reversi concept (custodian flip in Othello requires bracketing on a line, not the 2-friend outnumber pattern). The specific moore-26 + outnumber-2 interaction was the decisive tactic, and it's not a transfer from any single classical game.
- The **3D escape-lane** observation in Game 3 (build at z=2 to dodge row-trap) is also genuinely 3D-specific in that it uses the extra axis for parallel-distance buffering. In 2D Bridg-It-with-capture you'd only have y as escape axis, which constrains the attacker too.

**Verdict:** The combination is **modestly novel** — the game has a distinct tactical primitive (row trap → double capture) emergent from moore-26 + outnumber-2, which I have not seen written up. But the strategic frame is straightforward "connection race with capture overlay." An expert in connection games (Slither, Bridg-It, Hex) would transfer 80% of strategy on day one. The 3D substrate adds escape-axis tactics but not fundamentally new ideas. **Novelty 5/10.**
