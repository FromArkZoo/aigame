# Team-21 Evaluation: Game `9841cb9e6750`

**Run 17 rank #4** (GE 0.2779, ELO 2593) — the only top-10 R17 game with a **threshold** win condition rather than connection. 3D grid 4³, surround capture, place-only, alternating, influence propagation.

---

## PHASE 1 — RULE COMPREHENSION

### Board
- **Topology:** 3D **grid**, axis_size = 4 → **64 cells**, von-Neumann adjacency (max degree 6 in interior, 3 at corners).
- **Action space:** 65 = 64 placements + 1 pass. **Place-only**, no movement actions.
- **Cell index** = `z·16 + y·4 + x`. Easy to typo; pilot helper warning is justified.

### Turn structure
- **Alternating**, 1 piece per turn, P1 starts. Two consecutive passes → draw.

### Placement
- Place on any **empty** cell. No suicide / ko restriction at the placement gate (super-ko is global, not relevant here because positions never repeat in a 25-stone game).

### Capture — surround (Go-style)
- After placing, every adjacent enemy group is checked; groups with **0 liberties** are removed.
- `capture_rule.threshold = 1` is unused by `_capture_surround` (only outnumber consults it). So the rule is plain Go capture.

### Influence propagation (the engine that actually drives the game)
- After capture, the placed stone broadcasts a value field: every cell within Manhattan radius 2 receives `sign · 1.4027 · 0.4724^d`, where `sign = +1` (P1) or `−1` (P2).
- Numerically: own cell **±1.4027**, d=1 cell **±0.6628**, d=2 cell **±0.3131**.
- Values clamp to ±100. They persist on cells even if the stone is captured (but a cell only contributes to a player's score while *that* player owns it).

### Win condition — **threshold race**
- `condition_type = "threshold"`, `threshold = 42.080`, `max_turns = 100`.
- After each move, the engine sums `board_values[c]` over each player's owned cells. P2's sum is negated. If a player's effective sum exceeds 42.08, they win. If both cross same tick, higher margin wins; FP-tied → draw (R17 tolerance fix).

### Degeneracy / inertness flags
1. **Surround capture is effectively inert in competitive play** (Phase 2 evidence: zero captures in 3 full games × 22-25 plies). Interior stones need 6 surrounding enemy moves to capture; in a threshold race that ends in ~23 plies, no group is ever short on liberties.
2. **The `move_constraint: "adjacent_empty"` field is dead** — `action_types = ["place"]` so movement is never enabled.
3. **`target_dimension`/`target_dimension_p2` fields exist but are unused** (only consulted for `connection` win condition).
4. **Threshold is reachable** — empirically a tight 2×2×2 P1 cube alone gives 34.6 (sub-threshold), and adding 2 face-attached stones gives 45.2 (super-threshold). So the game does *not* resolve as a double-pass draw; it converges through threshold.

### Per-side strategy guides (from Phase 2)

**P1 strategy**: open in any cell with ≥4 neighbors (interior best, but face works), then on every subsequent move compute the "swing" of each empty cell and play the highest. The dominant motif is the **"valley fill"**: cells at distance 1 to multiple own stones — these compound the cluster's value at +5 to +7 per move.

**P2 strategy**: identical to P1's, but starting one tempo behind. P2 must play the symmetric/mirror move to match P1's swing. P2's best practical chance is to find a triple-d=1 valley before P1 does (Game 1 turn 22 was the closest moment), but with optimal P1 play, P1 always finds a response.

### Player reflections

**P1 reflection**: I never felt threatened. The 1-tempo lead converted directly into a threshold-race lead. Best surprises were the +6.5 to +7.2 swing cells (5-6 stones in radius) — those occasionally cause the lead to oscillate but never invert decisively. I wouldn't change my approach: maximize d≤2 own-stone neighbor count.

**P2 reflection**: I tried twice to overtake (Game 1 move 22, Game 2 move 16). Both times P1 had a +5+ counter that I couldn't prevent. The only way I could have won is if P1 had skipped a tempo (passed or played terribly). The mathematical "swing symmetry" — contesting and extending give the same own-eff gain — means I can't actually disadvantage P1 by my choices; I can only delay.

Endgame: All 3 games reached the threshold cleanly. None hit max_turns. None drew. None had captures.

## PHASE 3 — JOINT STRATEGIC ANALYSIS

### Are there distinct viable strategies?
**No.** Every game converges to the same dominant strategy: "maximize own d≤2 neighbor count via valley fills." We tried face-cell openings, corner openings, and contesting plays — all collapse to the same threshold-race endgame within 1-2 turns of each other.

### Meaningful counter-play?
**Almost none.** P2's options are essentially: (a) mirror P1, (b) contest in P1's heavy zone (mathematically equivalent to mirror — see "swing symmetry" below), (c) attempt capture (impossible — see Phase 1 inertness).

### Short-term vs long-term tension?
Negligible. Every move's value is immediately scored; there's no hidden delayed payoff. The only "tactical" element is finding triple-d=1 cells before the opponent.

### Emergent concepts
- **Cluster compounding**: Stones near other own stones reinforce each other's values. Tight cubes outscore spreads.
- **Valley fill**: A cell at d=1 to 2+ own stones gains +5+ swing vs +3.96 for a single-d=1 extension. This is the only emergent structural concept.
- **Swing symmetry** (mathematical observation): For any 1-1 contest where P1 places a stone in a cell already touched by P2, the net "swing" advantage is identical to placing in an untouched cell. P1's loss of own value (P2's −Δ accumulated on the cell) exactly equals P1's gain from depriving P2 of that influence. This is a striking but *unhelpful* symmetry — it means contesting is strategically identical to extending.
- No territory, ko, sente, mutual annihilation, or 3D path-routing emerges.

### Does 3D enable anything 2D wouldn't?
**No.** The 4³ cube has 64 cells; any 2D 8×8 grid (also 64 cells) under identical influence/threshold rules would produce the same dynamics. The 3D structure provides slightly more 6-neighbor interior cells (8 vs 4 in 2D 8×8), which marginally accelerates cluster-value compounding, but doesn't enable 3D-specific concepts like "z-axis pillars," "volumetric encirclement," or "skew-plane connection." Captures are dead, so the higher 3D connectivity (which would matter for capture mechanics) is wasted.

### Topology variation impact
With this same rule set, hex/torus/moore would change capture-vulnerability and influence radius coverage but the threshold race structure would be identical. The R17 generator's choice of 3D grid is a non-event for the strategic experience.

### Movement?
**Not applicable.** `action_types = ["place"]`. Move count across 3 games: 0/0 (place-only). The `move_constraint` field is vestigial.

### First-mover advantage (quantified)
- **3 / 3 games won by P1.**
- Game 1: optimal P1 opening, win on turn 23.
- Game 2: handicapped P1 (face opening), win on turn 25.
- Game 3: severely handicapped P1 (corner opening, only 3 neighbors), still win on turn 25.
- Even with the worst possible P1 first move, **P1's 1-tempo lead is sufficient to win**.
- This is a **decisive structural P1 bias** — the "seat-balance floor" gate (R17 hard-zero at seat_bal < 0.1) appears not to have caught this game, presumably because the *trained PPO agents* lose tempo to sub-optimal play and produce more balanced winrates than optimal play. With optimal play, this game is essentially solved P1.

## PHASE 4 — NOVELTY ADVERSARY (mandatory)

### Adversary case: this is unoriginal

**(a) Catalog comparison.** The threshold-race-with-influence formula is reminiscent of:
- **Reversi/Othello** — both accumulate territorial value; Reversi flips stones, this propagates value. Surround capture replaces flipping.
- **Go** — surround capture *is* Go's capture rule, threshold replaces territory counting.
- **Hyperborea / Fields of Arle / influence games** — radial influence with decay is well-established in board-game design.
- **3D Tic-Tac-Toe / Qubic** — same 4³ board but linear-line wins, not threshold. **3D Connect-4** ditto.
- **3D Hex / Y-3D** — connection wins in 3D; this game ignores connection entirely (despite the engine supporting it; threshold is the chosen rule).
- **Tumbleweed** — influence-based stone strength, closest mechanically. But Tumbleweed has discrete stones, this has continuous fields.

**(b) CA literature.** Not applicable — game uses classic capture, not CA.

**(c) Simultaneous-game catalog.** Not applicable — alternating.

**(d) "3D Connection Go with custodian" claim.** Not applicable — this game is 3D *threshold* race, not connection. The novelty hypothesis the run-context calls out is for `44f6630277b3` (the champion), not this game.

**(e) Re-skin under topology transform?** Yes, plausibly. This game = "Reversi value accumulation + Go surround capture + Gaussian-ish influence kernel + threshold tripwire." A 2D 8×8 version would play identically. The 3D coordinate is a coordinate transformation; nothing strategic depends on z.

**(f) Expert transfer?** A Reversi player with Go capture experience would pick this up in 5-10 minutes. The "valley fill" insight is reachable in one game. There's no non-obvious tactical or strategic layer.

**Adversary's headline claim**: This is **Reversi territorial accumulation + Go-style surround capture (which never fires) + radial influence on a 3D grid**. The threshold race format is a thin re-skin of "first to N points wins" common to many gameplay loops. The 3D substrate doesn't generate any novel strategy.

### Player rebuttals

**P1 rebuttal**: The "swing symmetry" math (contesting and extending give the same own-Δ) is genuinely unusual — it's not a feature of Reversi or Go. The radial-decay influence with explicit decay-coefficient produces value-fields that aren't standard. **But** — these features don't generate novel *strategy*, only novel mechanics. The strategy collapses to the same "valley-fill" formula regardless. The mechanic is novel; the play isn't.

**P2 rebuttal**: The fact that surround capture is effectively dead (zero captures in 22-25 ply across 3 games) means this is *not* a Go variant in practice; it's an influence-accumulation game with a vestigial Go-capture rule. That removes the strongest case for "this is just 3D Go." On the other hand, that emptiness *is* a flaw, not a feature.

### Novelty score (post-rebuttal): **4/10**
- Mechanically unusual (radial influence + threshold race + dead surround capture).
- Strategically reduces to a known pattern (Reversi-like territory accumulation, no captures, hardcoded P1 win).
- The 3D substrate is a coordinate transformation, not a strategic addition.
- Closest analog: **a continuous-influence Reversi variant on a 4³ grid**, where stone-flipping is replaced by influence-field accumulation and the win is a threshold tripwire instead of full-board control.

## PHASE 5 — VERDICT

**Team ID:** team-21
**Game ID:** 9841cb9e6750
**Rules Summary:** 3D 4×4×4 grid, alternating place-only, surround capture (inert), radial influence propagation (radius 2, decay 0.47), threshold win at 42.08 effective board-value sum.
**Topology:** 3D grid, axis 4 (64 cells), von-Neumann adjacency.
**Turn Structure:** Alternating.
**Hybrid actions:** No (place only — `action_types: ["place"]`).

### SCORES (1-10)

- **Strategic Depth: 3/10** — Optimal play reduces to a single mechanical procedure ("compute swing of every empty cell, take the max"). No deferred payoffs, no read-ahead requirements beyond 1 ply, no opponent modelling, no hidden information. The dominant strategy is found on game one.

- **Emergent Complexity: 3/10** — Exactly one emergent concept arises from the rule set: the "valley fill" (cells with 2+ own d=1 neighbors give outsized swing). No territory, ko fights, life-and-death, sente/gote, or volumetric tactics emerge. The "swing symmetry" math is a curious property but a strategic dead-end.

- **Balance: 1/10** — **3/3 games won by P1**, including a game where P1 was deliberately handicapped to a corner opening with only 3 neighbors. The 1-tempo first-mover advantage is decisive under optimal play; trained PPO agents likely win-rate-balance only because they make tempo-losing mistakes. The R17 seat-balance hard-zero gate (< 0.1) presumably did not fire because PPO winrates were closer to 50/50 than optimal play would suggest.

- **Novelty (post-adversary): 4/10** — Mechanically unusual combination (radial-decay influence + dead surround capture + threshold race) but strategically reduces to "first-mover Reversi-like accumulation." 3D is a coordinate dressing.

- **Replayability: 2/10** — Every game converges to the threshold race in 23-25 plies with the same underlying motif. Different openings produce a 1-2 turn delta in the resolution but the same outcome (P1 wins). I could not find an opening that flips the result.

- **Overall "Would I play this again?": 3/10** — As a curiosity, perhaps once to confirm the dynamics. As a competitive game, never — the result is decided by who moves first.

### CLOSEST KNOWN-GAME ANALOG
**Continuous-influence Reversi on a 3D grid** with vestigial Go-capture and a threshold tripwire. Unlike Reversi, stones don't flip — instead, every placement broadcasts a Gaussian-decay value field, and the first player whose owned-cell field sum exceeds 42 wins. The Go-capture rule is present but never fires in equilibrium play.

### KILLER FLAWS

1. **Severe first-mover advantage.** P1 wins 3/3 with optimal play, even when handicapped. The threshold race always favors the player who starts one tempo ahead, and no defensive or aggressive P2 strategy can recover the tempo.
2. **Surround capture is inert.** Zero captures across 67 plies of play. The capture rule is bait — it advertises Go-like dynamics that the game cannot deliver because clusters always have abundant liberties when the threshold trips.
3. **Vestigial action fields.** `move_constraint: "adjacent_empty"` is set despite `action_types: ["place"]` — leftover from mutation history. No effect on play but indicates rule generation isn't fully consistent.
4. **Mechanical dominant strategy.** "Compute swing for every empty cell, play the max." A 1-line greedy heuristic finds optimal moves; there is no horizon beyond 1 ply.
5. **Ignored degrees of freedom.** The engine supports super-ko, capture, multi-turn pieces, etc., none of which engage in this game's equilibrium.

### BEST QUALITY
**The radial-decay influence kernel.** The `strength · decay^d` field with `strength=1.40, decay=0.47, radius=2` produces clean per-distance contributions (1.40 / 0.66 / 0.31). The math has a pleasing symmetry between own-cluster reinforcement and enemy-influence dampening. As a *mechanic*, it's elegant; the game just doesn't extract strategic depth from it.

### 3D STRUCTURAL CONTRIBUTION
**None of substance.** A 2D 8×8 grid with the same rule parameters would produce mechanically and strategically equivalent play. The 4³ cube provides 8 interior 6-neighbor cells (vs 4 for 2D 8×8), which marginally accelerates cluster compounding but adds no qualitative depth. The 3D substrate is a coordinate transform of an 8×8 game with adjacency rewritten. This game does **not** support the run-17-thesis hypothesis that the 3D shift is genuinely strategic — at least, not for this threshold-race variant. (The hypothesis may still hold for the connection-win cousins like rank-1 `44f6630277b3`.)

### IMPROVEMENT IDEAS

The single highest-impact change: **asymmetric thresholds** — give P2 a lower threshold than P1, e.g. P1 needs 45.0 and P2 needs 38.5. Calibrate so that under optimal play the per-turn race is symmetric. This would directly address the killer first-mover advantage without changing any other mechanic. Alternative ideas, in decreasing priority:

- **Make captures meaningful**: each captured stone's `board_value` accrues to the capturer's threshold sum (not just removed). Would create incentive to attack groups.
- **Increase radius to 3 or strengthen decay so distant cells matter**: would create more positional tension between clusters.
- **Reduce threshold by 30%** (to ~30): games end earlier, before optimal play stabilizes; might preserve more variance.
- **Ban placement on cells already heavily influenced by enemy**: would force genuine territorial planning rather than mathematical valley-fill.
