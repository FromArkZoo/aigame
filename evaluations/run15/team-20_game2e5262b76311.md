## PHASE 1 — RULE COMPREHENSION

### Board structure
- 8x8 board, 2D, **torus topology** (both axes wrap — (x, y=0) is adjacent to (x, y=7); (x=0, y) is adjacent to (x=7, y)).
- 64 cells + 1 pass action (action 64). Total 65 actions.

### Turn structure — **SIMULTANEOUS**
- Both players submit their action in the same round.
- **Collision rule**: if both players select the same non-pass cell, mutual annihilation — neither stone is placed; cell stays empty.
- Steps_per_turn=2, and in R15 the CA loop runs BOTH P1's and P2's perspectives each tick (so the CA is applied 4 times per round: P1, P2, P1, P2).

### Action types
- Place only (no move).
- Placement constraint: `adjacent_to_own` with `first_move_anywhere=True`. A player's first placement (more precisely, any placement while the player has 0 stones on the board — which can recur after total annihilation) can go anywhere; subsequent placements must be adjacent to an already-owned cell.
- Pass is always legal.

### Capture / CA
- `capture_type=none` — no direct capture.
- CA transition table has 75 entries; after subtracting identity maps, the **non-trivial** (non-identity) entries are:
  - `0,1,2 → 1` (empty cell with 1 friendly + 2 enemy neighbours is **born friendly**)
  - `0,2,1 → 1` (empty cell with 2 friendly + 1 enemy → born friendly)
  - `1,0,2 → 0` (friendly with 0 friendly + 2 enemy **dies**)
  - `1,1,3 → 0` (friendly with 1 friendly + 3 enemy dies)
  - `2,2,0 → 0` (enemy with 2 friendly + 0 enemy dies — the swap of `1,0,2`)
  - `2,3,1 → 0` (enemy with 3 friendly + 1 enemy dies — the swap of `1,1,3`)
- Symmetry verified: T(1, f, e) = swap(T(2, e, f)) and T(0, f, e) = T(0, e, f) both hold.
- Semantically this is a mild "birth on 1f+2e or 2f+1e" and "death when heavily enemy-surrounded" rule — similar in spirit to a minority-birth / majority-kill Life variant. Only 6 of 75 entries are non-identity; CA is sparse but not trivial.

### Propagation / influence
- `prop_type=none`. No influence propagation. `board_values` is always zero and threshold field is unused for this game (connection wins instead).

### Win condition — **CONNECTION** (Hex-like)
- `condition_type=connection`, `threshold=0.5` (ignored for connection), `target_dimension=1` for P1, `target_dimension_p2=0` for P2.
- P1 wins by connecting y=0 to y=7 through owned cells.
- P2 wins by connecting x=0 to x=7 through owned cells.
- Connection is checked via BFS on the cell set restricted to owned cells, using topology neighbours (which wrap on torus).
- **CRITICAL FINDING**: on an 8x8 torus, y=0 and y=7 are **directly adjacent** (they wrap). A single cell (c, 0) and a single cell (c, 7) in the same column ARE neighbours. So P1 can satisfy the win with **2 stones** placed on the same column: one at y=0, one at y=7. Same for P2 across x.
- `max_turns=100`.

### R15 behaviour
- Double-pass → DRAW (winner=None).
- `_check_connection` iterates `for player in (1, 2)` and returns on the first player whose cells connect. If both players satisfy the connection on the same tick, **P1 wins the tie by iteration order**. This directly applies to this game because both players can execute a 2-move wrap-win plan and both complete it on Round 2.

### Degeneracy flags
- **Torus wrap + connection + axis_size 8** = trivial 2-move win for each player. Both a P1 and a P2 trivial win exist; under symmetric racing P1 always wins via check-order iteration.
- CA is effectively dormant for the first ~4 placements (not enough density for the birth rule to fire; death rules need surrounding enemies which don't yet exist).
- The stated threshold (0.5) is inert because the win condition is connection, not threshold.

### Strategy guides

**P1 strategy guide**: Open on any y=0 cell; on R2 place the same column at y=7 to wrap-close the connection. ## PHASE 3 — STRATEGIC ANALYSIS (joint)

- **Distinct viable strategies?** No. One strategy dominates: "place at (c, 0) on R1, (c, 7) on R2, for any c." The asymmetry is only in the axis (P1 uses y, P2 uses x). There is no incentive to develop pieces in the interior — the interior is irrelevant to both win conditions. CA never meaningfully engages because games end on R2.
- **Meaningful counter-play?** Very limited. Collision-blocking is a matching-pennies subgame against the racer's column choice, expected to succeed with probability 1/8 per round. Positional "block-and-build" play becomes possible once both wrap lanes are closed, but P1 can always force a race by simply not collision-attempting P2 — and winning the race via iteration.
- **Short-term / long-term tension?** None. The game ends before long-term considerations can accrue. No sacrifice mechanic exists.
- **Emergent concepts?** Almost none. The most interesting strategic concept is "column commitment timing" — revealing your target column as late as possible to reduce collision risk — but this only helps when the opponent is actively collision-guessing. Mutual annihilation (simultaneous-specific) appears but serves only as a retry mechanism, not a strategic tool.
- **Topology matters?** Yes — but in a degenerate way. Torus wrap is the entire game. On a grid topology, connection would require 8 pieces across the board (a real Hex-like game). The wrap collapses that to 2, and that collapse trivializes the game.
- **First-mover advantage?** P1 wins 3/3 games including the Game-3 seat swap, entirely because of engine iteration order in `_check_connection`. The simultaneous mechanic did NOT neutralise seat bias — it introduced a brand new one via the tie-break rule. Quantified: 3 out of 3 games → P1 win rate = 100%.

## PHASE 4 — NOVELTY ADVERSARY (MANDATORY)

### Adversary arguments

**(a) Comparison to known abstract strategy games**:
- **Hex**: this is fundamentally a Hex variant — two players, connection win, opposing sides for each player. The only structural differences are torus wrap (which makes connection trivial) and the simultaneous turn. Strip those out and you have literal Hex.
- **Y / Havannah**: same connection family, different goal topology. The torus-wrap win here isn't any of Y/Havannah's shapes but is simpler than either.
- **Connect6 / Gomoku / Pente / Reversi / Amazons / Lines of Action**: no resemblance — those are capture/n-in-row games.
- **Mancala**: not relevant — no sowing.
- **Tumbleweed / Slither**: no influence / shadow mechanics (propagation is off here), so no real overlap.
- **Chameleon** (connection on torus): actually Chameleon is played on a hex-like torus board, and the connection-on-torus framing exists in the hobbyist literature — but Chameleon has its own colour-changing mechanic absent here.

**(b) CA literature**:
- The CA is a 6-non-identity-entry 3-state (0 / friendly / enemy) rule with max_neighbours=4. Life-like CAs (B/S notation on two states) aren't directly comparable, but the closest analogue is a weak minority-majority birth rule with an enemy-surround death rule. It is not Conway's Life, HighLife, Day&Night, or Immigration Game — but it is also barely a CA because it fires only when cells are densely contested. Given games end on round 2, the CA is effectively dormant. It is *not* a known named rule, but it is also *not* meaningfully exercised in actual play.

**(c) Simultaneous games**:
- **Diplomacy order resolution**: Diplomacy's support/move adjudication has nothing in common beyond "both submit orders at once." No diplomatic support here.
- **Colonel Blotto / Blotto**: resource allocation across fronts. Not relevant — placements aren't a budget.
- **Simultaneous Go / Gungo / Pair Go**: Pair Go is turn-alternating between 4 players, not simultaneous. Gungo-style mutual-annihilation is the actual closest analogue — same collision-on-same-cell rule. But Gungo is still a territory-capture game, not a connection game.
- **Matching Pennies / RPS-scaled**: the collision-block sub-game is exactly matching pennies over 8 columns. This does play a micro-role.

**(d) Re-skin hypothesis**:
The most damning adversary claim: this is **"Hex on an 8x8 torus with simultaneous moves and free first move"**. The CA is inert, the propagation is off, the capture is off, the threshold field is unused. Strip them and you have Torus-Hex-Simultaneous, which reduces further: because of wrap, both players' connection requirements collapse to "two stones on opposite faces of their axis" — which in turn reduces to **a 2-move race with a single chokepoint (the opponent's collision guess)**.

**(e) Expert transfer test**:
A Hex expert would gain very little advantage because the torus makes the game trivially about wrap-around wins rather than about connection paths through the interior (which is where Hex expertise lives). A Matching-Pennies / poker expert would gain slightly more value (the collision sub-game). A CA expert gains nothing (CA never activates in typical play). Net: an expert at Hex would *not* have an immediate advantage — they'd miss the wrap trick at first, then learn the "real game" is a 2-round race, which is trivial once seen. So the game is neither a true Hex challenge nor a novel composite with genuine depth.

### Rebuttal (P1/P2 jointly)

- Claim: the game is nominally a composite of (connection + torus + simultaneous + adjacent_to_own + CA). The composite is real — no single known game combines all five features. But this is a weak defence because:
  - The CA is inert in actual play.
  - The adjacent_to_own constraint is bypassed on round 1 by `first_move_anywhere` and is irrelevant on round 2 (the wrap cell is always adjacent).
  - The propagation is literally turned off.
  - The simultaneous mechanic's only strategic expression is the collision-block sub-game, which is Matching Pennies.
- Concrete Phase-2 moments where a known-game analogy would have failed: **none that we observed**. Every decision reduces to either "race" or "guess a column to collide." Both are already in Matching Pennies or Hex-on-torus.
- Honest verdict: the game is a thin torus variant of a connection-race with a simultaneous twist that collapses to Matching Pennies on the race column. The only real novelty is the composite of "torus + connection + simultaneous + adjacent_to_own + first_move_anywhere" — but the composite doesn't produce emergent depth; it produces a degenerate 2-move race.

**3/10.** The game survives the "this is Hex" challenge because torus-wrap and simultaneous play change the tactical unit radically; it does not survive the "this is a 2-move race" challenge. It is not a re-skin of a single known game, but nor is it meaningfully novel as a playable artifact.

### CLOSEST KNOWN-GAME ANALOG
**Hex on an 8x8 torus** (with simultaneous turns). Not identical because (a) torus wrap trivialises the connection to 2 moves, (b) turns are simultaneous with mutual-annihilation collisions, (c) the inert CA and propagation rules are present but unused. Still, a Hex player would recognise the spirit; the meaningful strategic content is much thinner than Hex.

### KILLER FLAWS
1. **Torus + connection + axis 8 → 2-move trivial win**. Any player can place (c, 0) then (c, 7) — directly adjacent via torus wrap — for an instant connection. This is the dominant strategy and all meaningful play evaporates into the R1-R2 window.
2. **P1 iteration-order bias in `_check_connection`**. When both players complete the wrap race on the same tick (the common case under symmetric optimal play), the engine iterates `for player in (1, 2)` and returns on P1. Observed 3/3 times in Phase 2. This flaw directly contradicts the claim that the simultaneous turn structure neutralises seat bias.
3. **CA is dead**. Games end at step 2; CA requires local density to fire. The 6 non-identity entries never trigger in real play.
4. **Propagation and threshold fields are off/unused**. Both are part of the rule schema but have no effect.
5. **`adjacent_to_own` is effectively bypassed**. `first_move_anywhere=True` lets R1 go anywhere; R2 only needs to be adjacent to the single R1 stone, which on torus includes the wrap cell on the far side. Constraint never bites.

### BEST QUALITY
The one genuinely interesting moment is the **collision sub-game on R1**: does P2 try to guess P1's column to annihilate it? This is a clean micro-instance of Matching Pennies embedded in a placement game, with an asymmetric payoff (P1's "gain" is "I still have first_move_anywhere and burned a round" vs "I win"). This mechanic *could* be interesting in a deeper game with more rounds.

### IMPROVEMENT IDEAS
**Remove torus wrap** — or switch the topology to grid/hex — so that connection actually requires building a path of ~8 cells. This alone would re-engage the `adjacent_to_own` constraint (you'd have to *build* your path), activate the CA (enough density to trigger birth/death), and transform the game from a 2-move race into a genuine Hex-with-simultaneous-moves, which has novelty. Secondary improvement: fix the `_check_connection` iteration-order bias by declaring same-tick mutual connection a draw, so the simultaneous mechanic genuinely removes seat bias.
