## Phase 1 — Rule Comprehension

### Board & Mechanics
- 8x8 grid on **torus** topology (all edges wrap). 64 cells + 1 pass action = 65 actions.
- **Turn structure: SIMULTANEOUS.** Both players submit an action per round. Collision rule: if both players place on the same cell, both stones annihilate (cell remains empty). Resolved via `step_simultaneous()`.
- **Actions**: place only (no movement).
- **Placement constraint**: `adjacent_to_own` with `first_move_anywhere: True`. First move of each player may be anywhere; subsequent moves must be orthogonally adjacent (N/E/S/W, with torus wrap) to at least one of the player's own pieces.
- **Capture**: none. **Propagation**: none. Board values stay zero; the threshold check is irrelevant — this is purely a connection game.
- **CA**: Yes. `steps_per_turn=2`, `max_neighbors=4`. Because R15 fires both P1 then P2 per step, **4 CA applications run per round**. Transition table summary:
  - **Birth**: empty → P1 (acting player) if own_n=1 & enemy_n=2 OR own_n=2 & enemy_n=1 (i.e. 3-neighbor cells with any mix)
  - **Death (P1-perspective self=1)**: to empty if own_n=0 & enemy_n=2, or own_n=1 & enemy_n=3
  - **Death (P2-perspective self=2)**: to empty if own_n=2 & enemy_n=0, or own_n=3 & enemy_n=1
  - All other entries are identity.
- **Symmetry audit of CA table**: fails the claimed empty-cell symmetry. `T(0,1,2)=1` and `T(0,2,1)=1`, i.e. with mixed friendly/enemy on an empty cell, the rule births the acting player's color. Because CA fires P1 first each tick, the cell is birthed as P1, and when P2's tick arrives the cell is no longer empty, so P2 gets no chance to birth it. **Empirical P1 bias in random play: 28 P1 wins vs 9 P2 wins over 50 random games (avg 7.7 rounds).** Flag this as a residual asymmetry — the engine fix to run both perspectives per tick does not neutralize birth asymmetry because P1 always fires first.
- **Win condition**: `connection`, threshold 0.5 (irrelevant for connection type), `target_dimension=1` for P1 (must connect y=0 face to y=7 face), `target_dimension_p2=0` for P2 (x=0 face to x=7 face). BFS over own-stone graph using torus neighbors.
- **Max turns**: 100. Double-pass ends as DRAW per R15 rules.

### Rule flags
- **KILLER FLAW #1 — Torus wrap trivializes the Hex-style connection win.** Because `connects_faces` uses torus neighbors for adjacency (wrapping), but the "opposite faces" are defined as coord=0 and coord=axis_size-1, a player can win in **2 moves** by placing at (any x, 0) and then (same x, 7) — these two cells are torus-adjacent via the y=7↔y=0 wrap. Engine-verified:
  - `rounds "3:32,59:33"` → P1 wins round 2 via (3,0)↔(3,7).
  - `rounds "27:32,35:39"` → P2 wins round 2 via (0,4)↔(7,4).
  - This completely bypasses the CA and the adjacent_to_own constraint (first move is anywhere; second move is anywhere adjacent to own piece, and the wrap-neighbor qualifies).
- **KILLER FLAW #2 — Iteration-order bias on simultaneous wins.** If both players achieve connection in the same round, `_check_connection` iterates `for player in (1,2)` and awards the win to P1. Engine-verified: `rounds "3:24,59:31"` gives P1 the win when both completed wrap-wins simultaneously. Compounds Flaw #1 into a structural P1 advantage.
- **MINOR FLAG — CA is functionally inert at game speed.** With 2-move forced wins and chains of ~8 moves in contested play, the CA rarely triggers birth/death conditions (neighbor counts of 2–3 are rare with isolated pieces). Even when the game is played defensively, CA rarely meaningfully changes the board state.
- **Propagation/threshold fields are dead.** The printed "Threshold=0.500 P1_own_sum=0.000" confirms no influence accumulates.

Round 10, P1 played (3,7)=59 — torus-adjacent to (3,0) via wrap — and won instantly (BFS connects (3,0) and (3,7) through the wrap edge, treating them as a path from y=0 to y=7).

**P1 reflection (Game 1):** I stumbled into the wrap-exploit by accident. The intended strategy of "build a vertical column from y=0 to y=7" was blocked at (3,6) by P2, but I realized (3,7) is torus-adjacent to (3,0), giving an instant BFS connection. In retrospect, I should have played the wrap exploit on move 2 of the game.

**P2 reflection (Game 1):** The "parallel wall" defence (rows 5-6 below P1's column) felt productive because it looked like I was extending toward x=7, but I completely failed to notice that P1 could win by wrap. I needed to either (a) race P2's own wrap, or (b) block (3,7) — the one cell that completes P1's wrap. Defensive play must include the wrap-cell of each column P1 has touched on y=0.

### Game 2 — wrap-exploit race

Moves: `3:59, 4:60, 5:61, 6:62, 7:63, 2:58, 1:57, 0:56`

P1 opens (3,0). P2 blocks (3,7). P1 keeps extending right on y=0; P2 mirrors by extending right on y=7 (both exploiting torus adjacency and blocking each other's wrap). After 5 rounds, P1 has (3,0)(4,0)(5,0)(6,0)(7,0) and P2 has (3,7)(4,7)(5,7)(6,7)(7,7). P1 wraps back left to (2,0)(1,0)(0,0); P2 mirrors to (2,7)(1,7)(0,7). **P2 wins at round 8 with a complete y=7 row from x=0 to x=7 (BFS from (0,7) wraps to (7,7) and back, spanning the full x-axis).**

**P1 reflection (Game 2):** The 2-move wrap win is unstoppable if played on round 2 before opponent responds. But once P2 blocks the natural wrap cell, the blocking chain P2 builds is itself a potential winning chain for P2. Any y=0 "broad front" strategy feeds P2's win. Correct play after the wrap is blocked: switch axis — build vertically through the interior of the board (fight a classical Hex game) rather than spamming y=0.

**P2 reflection (Game 2):** Blocking is good because the blocking row IS my winning condition. P2 blocks P1's wrap along y=7 and naturally completes the x=0→x=7 connection. However, if P1 stops feeding the y=0 race and pivots to a classical interior Hex, P2 must switch too.

### Game 3 — seat swap. I played P2 against an adversarial P1. Symmetric 2-move race.

Moves: `3:24, 59:31`

- Round 1: P1 plays (3,0)=3 (classical wrap opening). P2 plays (0,3)=24 (mirror wrap opening on the orthogonal face).
- Round 2: P1 plays (3,7)=59, P2 plays (7,3)=31. **Both players simultaneously complete wrap-connection wins.** The engine iterates players (1,2) in `_check_connection` and awards the win to P1.

**P2 reflection (Game 3):** Playing P2 with knowledge of the flaw, I cannot draw or win a symmetric race — the iteration-order tiebreaker gives P1 the win. My only counter is to refuse the race: use round 1 to block (3,7) instead of setting up my own wrap. That lets me survive to Game 2's scenario (which P2 won), but only because P1 cooperated in building the defensive row. An optimal P1 would pivot to interior play after its wrap is blocked, at which point I'm back in a classical Hex on torus with first-move advantage to P1.

**P1 reflection (Game 3, adversarial seat):** With the wrap-exploit known, opening (3,0) and following with (3,7) is the minimax play. If P2 blocks (3,7), I have lost zero information and P2 has committed a piece to defence. If P2 races, I win the tiebreaker. This is dominated strategy.

### Strategy guides

**P1 guide**: Open move 1 at (k, 0) for any k. Move 2: (k, 7). You win. If P2 blocks (k,7), pivot to a second wrap column: move 2 = (j, 0) for a different j; move 3 = (j, 7). P2 cannot block all 8 columns in time; eventually P2 is forced into an interior Hex game which favors P1 (first mover + CA birth bias + iteration-order tiebreak). Do not open in the interior of the board unless you have a reason to — it wastes a move.

**P2 guide**: Move 1: block the P1-wrap cell, i.e. if P1 opens at (k,0), play (k,7). This prevents the 2-move win. From there, you are playing a blocking/racing game. The good news: the blocking row you build across y=7 is itself a valid x-axis connection for P2 (via torus wrap), so blocking P1 = winning for P2. Do not open at (0, m) hoping to mirror-wrap — P1's tempo and tiebreak mean any symmetric race loses.

## Phase 3 — Strategic Analysis (joint)

- **Dominant strategy?** Yes. The wrap-exploit dominates all other openings. Under perfect play by both, P1 wins via the open-and-wrap (or wins the tiebreaker after defence). The "real" game only emerges in the narrow scenario where P2 blocks the wrap AND P1 builds a y=0 broad front instead of a new wrap column — and that scenario was P2's Game 2 win. Genuine strategic depth is essentially absent.
- **Counter-play?** Very limited. P2's only real option is Move-1 block, after which P2 is passively responding to P1's decisions. The CA never drives any meaningful state change in observed games.

- **Emergent concepts**: On paper, a Hex-on-torus with CA background and simultaneous placements could have produced interesting tempo, mutual annihilation, and racing dynamics. In practice, the wrap-win dominates everything.
- **Topology matters**: Yes, but pathologically. The torus is the entire game — on a non-torus grid, the connection game would be classical Hex-variant (and much deeper). Here, torus is the killer.
- **First-mover advantage quantified**: 2 of 3 games P1 won. Random play showed ~76% P1 decisive wins. Seat-swap in Game 3 did not rescue P2 (symmetric race → P1 tiebreak). Simultaneous play does NOT eliminate first-mover advantage here because the check-order tiebreaker hard-codes it.

## Phase 4 — Novelty Adversary

### Adversary case (NOT novel)

**(a) Known-game comparisons.**
- **Hex**: This game is literally Hex with orthogonal 4-neighbor adjacency on a torus, wrapped to allow 2-move wraparound wins. Hex without the wrap is identical in goal ("connect opposite faces with own stones"), minus the CA, minus the simultaneous structure. A Hex expert would read this as "Hex with a broken board."
- **Y**: Y-connection game on torus — same family. Connect-through-own-stones goal.
- **Gomoku/Connect6**: Not analogous — those are N-in-a-row, not face-to-face connection.
- **Go**: Connection and territory are core Go concepts; no direct match, but any Go player's intuition for "wall building" transfers immediately.
- **Life-like CA**: The CA transition table is not a known Life-like rule. It is closest to "Seeds" (B2/S0) in the birth rules (birth at 3 neighbors mixed) but the death rules are non-standard. It is also not Conway's Life, Day & Night, or HighLife. However, the CA is effectively inert because placements resolve the win in <10 rounds.
- **Tumbleweed**: influence-based territory, different mechanic entirely.
- **Slither**: movement-based, different.
- **Diplomacy**: simultaneous orders with collision-annihilation has a rhyme with Diplomacy's dislodgement, but trivially so.
- **Blotto/Colonel Blotto**: No resource allocation here.
- **Gungo (simultaneous Go)**: closest cousin — simultaneous placement with collision resolution — but Gungo uses Go win conditions (territory), not Hex-style connection.

**(b) CA literature.** Not a known Life-like rule. But the CA is practically inert given game speed; even if it were Conway's Life, it would not matter because the game ends before any CA epoch completes a generation of activity.

**(c) Simultaneous games.** Collision-annihilation is not novel (Diplomacy, many sim-game variants); the combination with Hex-style connection is novel in prior abstract-game literature to my knowledge — but only because the combination is pathological (the wrap trivializes it).

**(d) Re-skin argument.** This game is **Hex on a torus with orthogonal adjacency + simultaneous tempo + a decorative CA**. Strip the CA, strip the sim (use alternating), use grid not torus, and you have Hex-on-8x8-grid with orthogonal adjacency — a known dud (usually played on a rhombic hex board with 6-neighbor adjacency). On torus, this Hex-variant is degenerate: the wrap-2-move-win is analogous to Hex on a torus being generally considered broken in the abstract-games community. **This is Hex-on-torus with a trivial first-player win.**

**(e) Expert-advantage test.** A Hex expert would have an immediate, dominant advantage. They would find the wrap-win in 1 minute and play it. The CA adds no strategic consideration because placement speeds outrun CA evolution.

### Rebuttal (P1 and P2 joint)

Rebuttal from concrete Phase 2 moments:
- **Game 2 had a unique moment** when P2 realized its blocking row across y=7 was simultaneously its own winning x-axis connection (via torus wrap along x=0↔x=7 within y=7). A Hex expert would not have predicted that "blocking" = "winning" in this configuration, because classical Hex lacks the wrap. This is genuinely torus-specific.
- The **collision-annihilation rule** would occasionally matter in games where both players race wrap-cells — if (3,0) and (3,7) were simultaneously played by both, they'd annihilate, which no classical Hex expert would be prepared to reason about.
- However, **these novelties are trapped inside a degenerate structure**. The unique moments only emerge in the narrow "P2 blocks wrap" branch, which an optimal P1 avoids by pivoting to a second wrap column.

**Novelty score: 2/10.** The game is Hex-on-torus with Hex-style connection, decorated with an inert CA and simultaneous placement that is neutered by iteration-order tiebreak. The adversary wins.

## Phase 5 — Verdict

**Team ID**: team-21
**Game ID**: 2e5262b76311
**Rules Summary**: 8x8 torus Hex-style connection game with simultaneous placement, adjacent-to-own constraint, and a decorative CA. P1 connects y=0↔y=7; P2 connects x=0↔x=7.
**Topology**: 8x8 torus (wrap on both axes).
**Turn Structure**: Simultaneous (collision = mutual annihilation).

### SCORES (1-10)

- **Strategic Depth: 2/10** — 2-move forced win for P1 (and P2). The "game" collapses before any real positional play emerges. Even with optimal blocking, the contested scenarios resolve in under 10 rounds.
- **Emergent Complexity: 1/10** — The CA is functionally inert at observed game speeds (0 of 3 games saw a CA birth or death change the outcome). No emergent territory, tempo, or pattern. The one interesting emergent fact — "blocking row is winning row" — is a consequence of the torus, not the game's dynamics.
- **Balance: 2/10** — Seat-swap evidence from Phase 2: 2 of 3 games P1 win (including the one with seat swap, where both players completed the win simultaneously and iteration-order awarded P1). Random-play sample: 28 P1 / 9 P2 / 13 draws. The game is structurally P1-biased due to (a) CA birth rule firing P1 first for mixed-neighbor empty cells, (b) `_check_connection` iterating P1 first on simultaneous wins, (c) shorter wrap-win from top → bottom faces is symmetrically available but tiebreak favors P1.
- **Novelty (post-adversary): 2/10** — This is Hex-on-torus with an inert CA and simultaneous-play decoration. The single torus-specific insight (blocking row = winning row) is not enough to overcome the "Hex re-skin" critique.
- **Replayability: 1/10** — Once you know the wrap-exploit (5 minutes of play), there is no reason to replay. The decision tree is tiny.
- **Overall "Would I play this again?": 1/10** — No.

### CLOSEST KNOWN-GAME ANALOG
**Hex on a torus with orthogonal (4-neighbor) adjacency.** Not identical: Hex uses 6-neighbor hex adjacency on a rhombic board with no wrap. The torus + 4-neighbor variant is known in abstract-game literature to be degenerate, which is exactly what is observed here.

### KILLER FLAWS
1. **Torus wrap trivializes connection win** — both players have a 2-move win by placing at (k,0) and (k,7) (or (0,k)/(7,k) for P2). Engine-verified.
2. **Iteration-order tiebreak** — in a symmetric race where both players win on the same tick, P1 wins by code iteration order.
3. **CA birth asymmetry** — empty cells with 3 mixed neighbors are always birthed as P1, because CA fires P1's tick before P2's.
4. **CA is effectively dead weight** — game ends in under 10 rounds, CA rarely triggers.
5. **Connection threshold (0.5) and propagation are both configured but unused** — threshold never evaluated (only used for threshold-type wins), propagation has prop_type='none'.

### BEST QUALITY
The single genuinely interesting observation (surfaced only in Game 2): on a torus, a defensive blocking row along the opposite face can itself form a winning connection for the defender via wrap along the orthogonal axis. This is topology-specific and has no classical-Hex analog. It deserves to live in a different, non-degenerate game.

### IMPROVEMENT IDEAS
**Remove the torus wrap for `connects_faces` (use strict coord=0 vs coord=7 with non-wrapping adjacency for the BFS only).** This kills the 2-move wrap exploit. The game becomes Hex-variant-on-8x8 with simultaneous placement + CA background, which could actually be interesting — the CA would have time to evolve, collision-annihilation would create genuine sim-play tactics, and neither the iteration-order tiebreak nor the CA birth asymmetry would dominate because games would run long enough for other factors to matter. Alternatively: change the win condition to "longest chain reaches length 8" or a threshold condition over the (currently inert) propagation field, which would force longer games where CA and simultaneity actually matter.
